# DESIGN.md — Hello-CN

> 哈啰出行国内业务移动端设计系统，服务共享单车、助力车等核心出行场景的 B/C 端产品。

## Token 绑定声明

本文件是纯视觉样式规范，不包含组件定义。当外部 skill 或组件库引用本文件时，需遵循以下规则：

1. **组件的所有视觉属性必须从本文件的 Token 中取值**，包括但不限于：
   - 色彩 → 查 `2. Color Palette & Roles`
   - 字体族、字号、字重、行高 → 查 `3. Typography Rules`
   - 圆角 → 查 `4. Radius 圆角体系`
   - 投影 → 查 `5. Shadow 投影体系`
   - 间距、边距 → 查 `6. Layout 布局体系` 和 `7. Grid 栅格体系`
   - 图标 → 查 `8. Icon 图标规范`
   - 分割线与边框 → 查 `9. 分割线与边框规范`

2. **禁止组件内硬编码视觉值**。如果组件需要蓝色主按钮，应引用 `Branding-1: #0076FF`，而非自行写入 `#0076FF` 或任何近似色值。

3. **Token 优先级**：当组件库自带的默认样式与本文件冲突时，以本文件为准。

4. **引用方式**：外部 skill 在生成组件代码时，应先读取本文件对应章节的 Token 表，再将 Token 值注入组件属性。

## 1. Visual Theme & Atmosphere

### 1.1 共享设计原则

- **设计哲学：** 功能优先、克制清晰，以蓝色品牌基调贯穿全产品线，信息层级分明
- **品牌一致性：** C 端与 B 端共享同一套色彩 Token、字体族和圆角体系，确保品牌感知统一
- **视觉节奏：** 4px 基础栅格，所有间距、尺寸遵循 4 的倍数递增

### 1.2 哈啰骑行 App（C 端主端）

- **产品定位：** 面向大众用户的共享单车 / 助力车高频出行工具
- **核心场景：** 扫码开锁 → 骑行导航 → 结算支付，单次交互时长短、操作路径浅
- **设计调性：** 轻快、可信赖、零学习成本
- **信息密度：** 中低——首屏聚焦地图与核心操作，非关键信息逐层收敛
- **视觉关键词：** 大留白、圆角柔和、蓝白主调、地图沉浸、底部浮层操作
- **动效策略：** 状态切换使用 ease-out 200–300ms 过渡，骑行态地图保持 60fps 流畅渲染
- **参考产品：** 支付宝出行频道、高德地图、美团骑行

### 1.3 两轮共享运维 BOS（B 端运维移动端）

- **产品定位：** 面向兼职运维人员与全职城市管理者的共享两轮车运维管理移动平台
- **设计调性：** 专业沉稳、高效精准、状态清晰
- **信息密度：** 中高——兼职侧以任务卡片和地图为主，全职侧以数据概览和列表管理为主，关键状态前置，支持快速扫读

#### 兼职运维（一线作业）

- **典型场景：** 地图找车、扫码识别车辆作业、调度 / 换电 / 车品等任务履约链路、工单记录、收入查看、出勤打卡等
- **视觉关键词：** 地图沉浸 + 任务列表双模式、大点击热区、语义色强化任务状态、进度可视化
- **交互策略：** 户外强光场景需保证高对比度；操作按钮不小于 44×44pt；支持单手拇指热区操作；任务链路步骤化引导，降低认知负担

#### 全职管理（城市经营）

- **典型场景：** 城市经营数据概览、车辆资产管理、运维人员出勤管理、调度 / 换电 / 车品等任务下发与执行结果追踪等
- **视觉关键词：** 数据卡片 + 指标摘要、结构化列表、筛选器前置、状态标签体系、层级式下钻导航
- **交互策略：** 信息聚合优先，首屏呈现关键经营指标；列表支持多维筛选与排序；管理操作需二次确认防误触

#### 共性原则

- **双角色适配：** 同一框架下通过角色权限区分功能入口，视觉语言保持统一
- **参考产品：** 美团骑手端（兼职作业流）、美团商家端（全职管理流）、蜂鸟配送、菜鸟裹裹

## 4. Radius 圆角体系

> 来源：Hello-TaroUI-Kit-2026 Figma 设计规范

### 圆角梯度

| Token | 值 | 用途 |
|-------|-----|------|
| radius-2 | 2px | 极小组件圆角，元素高度 < 28px 时使用 |
| radius-3 | 3px | 极小组件圆角，徽标 / 标签等 |
| radius-4 | 4px | 28px 高度按钮，或三级小模块 |
| radius-6 | 6px | 仅限 32px 高的按钮使用 |
| radius-8 | 8px | 二级模块圆角（卡片） |
| radius-10 | 10px | 仅限 40px 高的按钮使用 |
| radius-12 | 12px | 面板场景，例如对话框 / 弹出层 / 选择器等 |
| radius-full | 9999px | 圆形组件 |

### 梯度原则

| 梯度 | 圆角 | 适用范围 |
|------|------|----------|
| 第一梯度 | 12px | 页面级圆角——底部浮层、弹窗面板的顶部圆角 |
| 第二梯度 | 8px | 模块级圆角——卡片容器、内容区块、按钮（对应高度） |
| 第三梯度 | 4px | 模块内圆角——卡片内的子元素、小型按钮、输入框 |
| 第四梯度 | 2px | 极小标签圆角——飘带徽标、微型标签 |

### 特殊规则

- **按钮圆角与高度绑定：** 28px → 4px，32px → 6px，40px → 10px
- **贴角标签：** 圆角需与所在容器的圆角保持一致（如容器 8px，贴角标签对应角也为 8px）

## 5. Shadow 投影体系

> 来源：Hello-TaroUI-Kit-2026 Figma 设计规范

### 向下投影

主要应用于组件内部或组件本身，是比较常规场景的用法。

| 层级 | 名称 | box-shadow | 用途 |
|------|------|------------|------|
| 一级 | 基础投影 | `0 2px 8px 0 rgba(17,17,17,0.04)` | 组件 hover 状态，例如表格和树拖动 |
| 二级 | 中层投影 | `0 4px 8px 0 rgba(17,17,17,0.08)` | 下拉组件，例如下拉菜单 / 气泡确认框 / 选择器等 |
| 三级 | 上层投影 | `0 2px 8px 0 rgba(17,17,17,0.04)` | 警示或弹窗组件，例如全局提示 / 消息通知等 |

### 向上投影

主要应用于底部导航或工具栏等。

| 层级 | 名称 | box-shadow | 用途 |
|------|------|------------|------|
| 一级 | 基础投影 | `0 -2px 8px 0 rgba(17,17,17,0.04)` | 组件 hover 状态，例如表格和树拖动 |
| 二级 | 中层投影 | `0 -4px 8px 0 rgba(17,17,17,0.08)` | 下拉组件，例如下拉菜单 / 气泡确认框 / 选择器等 |

### 阴影数值（研发参数）

| 场景 | 背景 | box-shadow |
|------|------|------------|
| 一级投影（常用于灰色背景） | `#FFFFFF` | `0 2px 8px 0 rgba(17,17,17,0.04)` |
| 二级投影（常用于白色背景） | `#FFFFFF` | `0 4px 8px 0 rgba(17,17,17,0.08)` |

### 场景应用

| 场景 | 投影方向 | 投影级别 | 示例 |
|------|----------|----------|------|
| Titlebar 标题栏 | 向下 | 一级 | 顶部导航栏底部阴影 |
| Card 卡片 | 向下 | 二级 | 内容卡片容器（白色背景上） |
| Half Screen Views 操作浮层 | 向下 | 一级 | 半屏弹窗顶部标题栏 |
| Bottom Bar 底部栏 | 向上 | 二级 | 底部操作栏顶部阴影 |

### Android 原生阴影（Elevation）

M2/M3 都是安卓系统规范，M3 是最新版本，安卓投影根据用户手机系统调用。

| 投影级别 | M2 Elevation | M3 Elevation |
|----------|-------------|-------------|
| 一级投影（常用于灰色背景） | 1dp | 1dp |
| 二级投影（常用于白色背景） | 3dp | 3dp |

- **设计稿尺寸：** 390 × 844px @1x
- **设计模板：** 以 iOS 为基准

### 空间布局

元素与元素之间的尺寸都必须是 4 的倍数，同时建议使用所提供的间距尺寸，在这个基本规则下剩余空间自适应。

| 模块间距 | 12px | 模块与模块之间的纵向间距 |

## 7. Grid 栅格体系

> 来源：Hello-TaroUI-Kit-2026 Figma 设计规范

### 栅格的组成

栅格布局由三个元素组成：列、水渠和边距。

| 元素 | 说明 | 值 |
|------|------|-----|
| 列（Column） | 采用 6 列栅格布局，页面主体内容需放置在列的覆盖区域内。列宽根据屏幕尺寸浮动，不是固定值，使内容可以适应任何移动设备屏幕尺寸 | 6 列，宽度自适应 |
| 水渠（Gutter） | 分隔列与列之间的内容 | 固定 16px |
| 边距（Margin） | 内容与屏幕左右边缘之间的空间，通常不放置内容，使页面主体内容更加聚拢 | 固定 12px |

### 间距

间距是关于如何在布局和组件中放置元素的一组规则，为保证屏幕实际展示效果，固定采用 4 的倍数进行设计，额外补充 0px、2px，满足实际使用需要。

| 间距值 | 间距值 | 间距值 | 间距值 |
|--------|--------|--------|--------|
| 2px | 4px | 8px | 12px |
| 16px | 20px | 24px | 28px |
| 32px | 36px | 40px | 44px |
| 48px | 56px | 64px | 80px |

### 间距适用场景

- **元素边距：** 指 UI 元素之间的空间，可以对元素进行垂直和水平的边距定义

### 元素尺寸

元素尺寸指组件元素的宽度和高度。某些组件（如标题栏、按钮等）仅定义元素的高度，宽度不做指定，宽度根据栅格布局而定义。

#### 导航栏区域尺寸

| 区域 | 高度 | 说明 |
|------|------|------|
| 状态栏（Status Bar） | 44px | 系统状态栏 |
| 导航栏内容区 | 44px | 标题 + 操作按钮区域 |
| 导航栏总高度 | 88px | 状态栏 + 内容区 |
| 返回按钮区域 | 36 × 36px | 左侧返回按钮热区 |
| 右侧操作按钮 | 36 × 36px | 搜索、更多等操作 |
| 操作按钮间距 | 12px | 右侧按钮之间的间距 |
| 公告栏 | 36px 高 | 导航栏下方通知条 |
| 公告栏与导航栏间距 | 12px | 导航栏底部到公告栏顶部 |
| 导航栏图标 | 24 × 24px | 图标实际尺寸 |
| 图标与导航栏边距 | 12px | 图标距导航栏左/右边缘 |

## 8. Icon 图标规范

### 图标来源

- **图标库：** BOS iconfont Symbol 图标库，接入方式与图标清单见 `references/component-styles/basic/icon.md`
- **风格：** 以 iconfont 图标资产定义为准；优先选用语义和状态匹配的图标

### 尺寸体系

| 尺寸 | 线粗 | 用途 |
|------|------|------|
| 24px | 2px | 导航栏图标、主操作图标、列表右侧箭头 |
| 20px | 1.67px | 列表项图标、Tab 图标、次要操作 |
| 16px | 1.33px | 辅助信息图标、标签内图标、输入框前缀 |
| 12px | 1px | 极小场景，标签内微型图标、气泡指示 |

> 线粗规则：以 24px / 2px 为基准，图标缩小时线粗同比缩小（24:2 = 12:1）。当精确值在 CSS/SVG 中不易实现时，允许四舍五入到 1.5px / 1.25px / 1px，优先保证视觉一致。

### 图标使用优先级

- 组件样式文件已指定图标时，使用该组件文件定义的 iconfont 图标。
- 组件样式文件未指定图标时，从 `references/component-styles/basic/icon.md` 的图标清单中选择语义最贴近的 iconfont 图标。
- iconfont 中没有完全匹配的图标时，可使用含义相近的 iconfont 图标占位，并在交付说明中标注该占位关系；不要引入外部图标库或自行绘制图标替代。

| 占位/提示图标 | gray3 | `#879099` |
| 禁用态图标 | gray4 | `#BCC4CC` |
| 反色图标（深色背景上） | anti | `#FFFFFF` |
| 品牌色图标 | brand | `#0076FF` |
| 链接图标 | link | `#0076FF` |

### 对齐方式

- 图标与文字采用**水平左右排列**为主（图标在左，文字在右，或反之）
- 图标与文字垂直居中对齐
- 图标与相邻文字的间距：4px（紧凑）或 8px（常规）

### 图标与文字搭配规则

| 文字字号 | 图标尺寸 | 示例 |
|----------|----------|------|
| 16px | 20px | 正文、列表内容搭配图标 |
| 14px | 16px | 辅助信息、次要文字搭配图标 |
| 12px | 14px | 小字辅助说明搭配图标 |

## 9. 分割线与边框规范

### 分割线

| 属性 | 值 | 说明 |
|------|-----|------|
| 粗细 | 0.5px（@1x） | 基于 390px 设计稿的一倍图尺寸 |
| 颜色 | gray6 `#E1E6EB` | 中性色体系中的分割线专用色 |

#### 使用场景

| 场景 | 说明 |
|------|------|
| 列表分割 | 列表项之间的横向分割线，通常左侧留出缩进（与文字左对齐） |
| 模块分割 | 页面内不同功能模块之间的分隔，通常通栏无缩进 |

### 边框

| 属性 | 值 | 说明 |
|------|-----|------|
| 粗细 | 1px | 组件边框统一 1px |
| 颜色 | gray5 `#D3DAE0` | 中性色体系中的默认边框色 |

### 分割线与边框的区分

| 维度 | 分割线 | 边框 |
|------|--------|------|
| 用途 | 分隔内容区域，表示信息层级 | 界定组件边界，如输入框、卡片描边 |
| 粗细 | 0.5px | 1px |
| 颜色 | gray6 `#E1E6EB`（较浅） | gray5 `#D3DAE0`（较深） |
| 形态 | 单条水平线 | 围合矩形 |

## 10. Do & Don't

> 基于以上设计规范总结的常见正误对照，供设计与开发对齐使用。

### 色彩

| Do ✅ | Don't ❌ |
|-------|---------|
| 主操作使用品牌主色 `#0076FF` | 使用其他蓝色近似值替代品牌色 |
| 按下态使用 Branding-0 `#0061D1` | 按下态仅降低透明度模拟深色 |
| 禁用态使用 Branding-2 `#C2DEFF` | 禁用态使用灰色替代品牌浅色 |
| 主要文字使用 gray1 `#111111` | 正文使用 gray3 `#879099` 等浅色 |
| 次要文字使用 gray2 `#5A6066` | 次要信息与主要文字使用同一色值 |
| 语义色严格按 4 级梯度使用（深/标准/浅/极浅） | 随意混用语义色的不同层级 |
| Error 用红色系、Success 用绿色系、Warning 用橙色系 | 用颜色以外的方式作为唯一状态区分 |
| 弹窗遮罩使用 `rgba(0,0,0,0.7)` | 遮罩透明度过低（如 0.3）导致背景干扰 |

### 字体

| Do ✅ | Don't ❌ |
|-------|---------|
| 中文使用 PingFang SC，数字使用 Roboto | 中文数字混用同一字体族 |
| 标题字重统一 600（Semibold） | 标题使用 400 或 700 等非规范字重 |
| 正文字重统一 400（Regular） | 正文使用 Semibold 导致视觉过重 |
| 严格使用规范字号（32/28/24/20/18/16/14/12/10） | 使用 15/13/11 等非标准字号（特殊场景除外） |
| 10px 仅用于飘带徽标和气泡数字 | 10px 用于正文或辅助信息 |
| 行高与字号配套使用（如 14/22、16/24） | 自定义行高导致文字间距不一致 |

### 圆角

| Do ✅ | Don't ❌ |
|-------|---------|
| 按钮圆角与高度绑定：28px→4px、32px→6px、40px→10px | 所有按钮统一使用同一圆角值 |
| 卡片容器使用 8px 圆角 | 卡片使用 12px 或 4px 等非规范值 |
| 弹窗/浮层使用 12px 圆角 | 弹窗使用 8px 或更小的圆角 |
| 贴角标签圆角与容器圆角保持一致 | 贴角标签使用独立圆角值导致视觉断裂 |
| 遵循梯度递减：页面 12px → 模块 8px → 模块内 4px → 标签 2px | 内层圆角大于外层圆角 |
| 极小元素（高度 < 28px）使用 2px 圆角 | 小标签使用 8px 等过大圆角 |

### 投影

| Do ✅ | Don't ❌ |
|-------|---------|
| 灰色背景上的卡片使用一级投影（4% 透明度） | 灰色背景上使用二级投影导致阴影过重 |
| 白色背景上的卡片使用二级投影（8% 透明度） | 白色背景上使用一级投影导致卡片无层次 |
| 底部操作栏使用向上投影 | 底部栏使用向下投影或无投影 |
| 导航栏使用向下一级投影 | 导航栏使用过重的投影（如 16px 模糊） |
| 投影颜色统一使用 `rgba(17,17,17,x)` | 使用纯黑 `rgba(0,0,0,x)` 或其他颜色作为投影 |
| Android 使用 Elevation（1dp/3dp）替代 box-shadow | Android 端硬编码 CSS box-shadow |

### 布局与间距

| Do ✅ | Don't ❌ |
|-------|---------|
| 所有间距使用 4 的倍数（4/8/12/16/20/24…） | 使用 5px、7px、10px 等非 4 倍数间距 |
| 页面左右外边距固定 12px | 外边距使用 16px 或 8px 等非规范值 |
| 模块间纵向间距 12px，子模块间 8px | 模块间距与子模块间距使用相同值 |
| 内容区预留导航栏 88px + 底部安全区 34px 的空间 | 内容被导航栏或底部安全区遮挡 |
| 设计稿基于 390×844px @1x 输出 | 使用 375px 或 414px 等非标准宽度 |
| 栅格 6 列 + 水渠 16px + 边距 12px | 自定义栅格参数导致布局不一致 |

### 图标

| Do ✅ | Don't ❌ |
|-------|---------|
| 使用 BOS iconfont 图标库；缺少精确图标时用语义相近图标占位 | 使用 emoji、外部图标库或自行绘制图标 |
| 图标尺寸与文字字号搭配：16px 文字配 20px 图标 | 图标与文字尺寸不匹配（如 12px 文字配 24px 图标） |
| 图标线粗随尺寸同比缩放（24px/2px 为基准） | 所有尺寸图标使用相同线粗 |
| 图标色彩引用文字色 Token（gray1–gray4） | 图标使用自定义颜色或硬编码色值 |

### 分割线与边框

| Do ✅ | Don't ❌ |
|-------|---------|
| 分割线使用 0.5px + gray6 `#E1E6EB` | 分割线使用 1px 或更粗导致视觉过重 |
| 组件边框使用 1px + gray5 `#D3DAE0` | 边框使用分割线的颜色（gray6）导致边界不清 |
| 列表分割线左侧留缩进与文字对齐 | 列表分割线通栏无缩进 |
| 分割线与边框区分使用（粗细不同、颜色不同） | 分割线和边框混用同一样式 |

### 交互与可用性

| Do ✅ | Don't ❌ |
|-------|---------|
| 可点击元素热区不小于 44×44pt | 点击热区过小导致误触或难以操作 |
| 操作按钮区域使用 36×36px 热区 | 图标实际尺寸等于热区尺寸（24px 图标无外扩） |
| 状态切换使用 ease-out 200–300ms 过渡 | 无过渡动效导致状态跳变，或动效超过 500ms |
| 管理类操作（删除、批量处理）需二次确认 | 危险操作一键直接执行无确认 |
| 户外场景（BOS）保证高对比度文字 | 浅色文字在强光下不可读 |
| 语义色 + 文字/图标双重标识状态 | 仅靠颜色区分状态（色盲不友好） |
