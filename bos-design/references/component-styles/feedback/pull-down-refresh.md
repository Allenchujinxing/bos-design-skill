# Pull Down Refresh 下拉刷新

## Purpose

页面级下拉刷新交互，用户在页面顶部下拉触发数据刷新。常用于列表页、信息流等需要手动更新数据的场景。

This file records the Figma visual style only.

## 视觉 Token 声明

本文件中的颜色、圆角、间距等视觉属性引用自搭配的 Design 规范文件（如 `references/design.md`）。当切换到其他视觉系统时（如 HelloRide、HelloSaaS），对应 token 的色值会变化，组件结构和行为不变。

---

## 交互状态

| 状态 | 说明 | 视觉表现 |
|---|---|---|
| 下拉中（pulling） | 手指下拉但未达到阈值 | loading 图标跟随下拉距离旋转，文字"下拉刷新" |
| 释放刷新（loosing） | 下拉超过阈值，松手即触发 | 文字变为"释放立即刷新" |
| 加载中（loading） | 已触发刷新，等待数据返回 | loading 图标持续旋转，文字"正在刷新..." |
| 完成（success） | 数据刷新成功 | 文字"刷新成功"，短暂停留后收起 |

---

## 容器

| Property | Value |
|---|---|
| 高度 | 自适应（下拉距离决定，最大 80px） |
| 背景 | 跟随页面背景色（gray8） |
| 位置 | 页面内容区顶部上方（负空间，下拉时露出） |
| 布局 | flex 垂直居中，水平居中 |

---

## Loading 图标

| Property | Value |
|---|---|
| 样式 | 4 圆点旋转动画（同 `feedback/loading.md` 的 dots-loader） |
| 尺寸 | 24×24px |
| 颜色 | Branding-1 |
| 下拉中行为 | 图标旋转角度跟随下拉距离（0→360°） |
| 加载中行为 | 持续旋转，1.8s/圈 |

---

## 文字

| Property | Value |
|---|---|
| 字体 | 12px / regular 400 / gray3 / line-height 18px |
| 与图标间距 | 8px（图标在上，文字在下） |
| 对齐 | 居中 |

---

## 阈值与动效

| Property | Value |
|---|---|
| 触发阈值 | 下拉 60px 后释放触发刷新 |
| 最大下拉距离 | 80px（超出后阻尼增大） |
| 收起动效 | 高度从当前值回到 0，duration 300ms，ease-out |
| 成功停留 | 显示"刷新成功"后停留 500ms 再收起 |

---

## Do Not Infer

- 不要在非列表页使用下拉刷新 — 仅用于有数据更新需求的页面。
- 不要让下拉刷新和页面滚动冲突 — 仅在页面滚动到顶部时才响应下拉。
- 不要省略加载中状态 — 必须有视觉反馈告知用户正在刷新。
- 不要让刷新成功后立即收起 — 需停留 500ms 让用户感知到结果。
