# Card Patterns（信息卡片范式）

> 卡片是 BOS 里高频的信息承载单元，跨页面复用（列表页、地图面板、搜索结果、弹层详情）。先由页面策略判断当前问题是否需要卡片；决定使用业务对象卡片后，再用本文件参考不同对象的信息组织范式。

## 通用设计策略

### 卡片信息分层原则

每张卡片的信息从上到下按以下层级组织：

1. **识别层** — 让用户一眼认出这是什么（类型标签 + 状态）
2. **核心信息层** — 这个对象最关键的标识信息（编号、名称、地址）
3. **业务数据层** — 辅助判断的指标数据（完成率、数量、电量等）
4. **操作层** — 用户可以对这个对象做什么（按钮、导航）

不是每张卡片都有全部四层，根据场景裁剪。

### 状态表达始终在右上角

卡片状态（待处理/进行中/已完成/已取消）统一放在第一行右侧，与左侧的类型标签形成对比。用户扫描列表时，左眼看类型，右眼看状态。

### 数据用网格排列，label 在上 value 在下

多个指标并排展示时，用等宽列网格（通常 4 列），每列 label 在上（小字灰色）、value 在下（大字黑色）。用户横向扫描对比数据。

### 地址信息带导航入口

凡是有地址的卡片，地址末尾都带「导航」操作入口。运维人员看到地址后的下一个动作就是去那里，不要让他再找导航按钮。

### 关键 ID 支持复制

编号、订单号等需要跨系统使用的 ID，旁边带复制图标或「复制」文字按钮。

## 数据统计卡片

**适用对象**：街道/网格的执行表现概览

**信息层级**：
1. 识别层：分组标签（执行表现）+ 统计时间 + 时间范围选择
2. 核心指标：总完成率（大字突出）+ 趋势 Tag（上升/下降/预警）
3. 构成公式：任务总数 = 调度 + 换电 + 车品（用公式表达构成关系）
4. 分项指标：3 列并排（调度/换电/车品完成率），每列带色条标识
5. 占比条：系统指派占比 vs 人工自建占比（可视化条形图）

**组织范式**：
- 总指标最大最突出，分项指标作为解释补充
- 构成公式用「= + +」表达数据来源，让用户理解总数怎么来的
- 预警状态时总指标变色（如红色），提示异常
- 「指标说明」入口放在右上角，不占主信息空间

---

## 站点/网格卡片

**适用对象**：街道、网格的运营数据概览

**信息层级**：
1. 核心信息：站点名称（加粗）+ 右侧箭头（可点击进入详情）
2. 基础统计：运维人数 | 网格数 | 任务数（用竖线分隔的横向排列）
3. 分割线
4. 指标网格：4 列 × 2 行（完成率行 + 作业量行），label 在上 value 在下
5. 无数据时显示「--」

**组织范式**：
- 名称和基础统计在分割线上方，是「这是哪里」
- 指标网格在分割线下方，是「这里表现怎么样」
- 两层信息用分割线明确区隔，用户先定位再看数据

## 调度任务卡片

**适用对象**：调度任务（投车/收车/堆积调度）

### 列表场景

用在任务列表中，信息密度适中。

**信息层级**：
1. 识别层：任务类型标签（调度 + 堆积调度）+ 右侧状态（待处理/进行中/已完成/已取消）
2. 点位信息：出/入标记（圆形标签 + 竖线连接）→ 站点名称 → 地址 + 导航 → 实际/计划量
3. 执行信息：执行运维 + 时间（创建/截止/完成/取消时间，根据状态变化）
4. 操作层：分割线 → 操作按钮（待处理有双按钮，进行中有单按钮，已完成无按钮）

**组织范式**：
- 出/入用圆形标签 + 竖线连接表达方向关系，视觉上形成路径感
- 指派任务在编号前加「指派」Tag，与非指派任务区分
- 操作按钮只在可操作状态出现，已完成/已取消状态不展示操作区
- 多点位任务（出+入）用竖线连接两个点位块，卡片高度相应增加

### 地图作业场景

用在地图页面中，用户看到这张卡片就要开始作业，信息更完整。

**信息层级**：
1. 来源标签 + 倒计时：「系统派单」标签 + 「剩 XX分XX秒 可领/可做」— 时间紧迫感驱动行动
2. 快捷操作：右上角「加调出点/加调入点」按钮 — 地图场景中的即时操作
3. 计划量摘要：「计划调出 12 辆 计划调入 12 辆」— 全局数量概览在点位信息上方
4. 任务类型 + 状态：堆积调度/出单调度 + 待领取/进行中
5. 点位信息：出/入标记 → 站点名称 → 地址 + 导航 → 实际/计划量
6. 附加操作提示：「可做保养/清洗（共10单）>」— 调入场景下的附加任务提示
7. 任务 ID + 复制：完整任务 ID，支持复制
8. 创建时间
9. 底部双操作入口：「车辆调度记录」+「任务操作记录」— icon + 文字按钮

**组织范式**：
- 倒计时放在最顶部最突出位置 — 运维人员在地图上看到卡片时，第一眼看到的是「还剩多少时间」
- 快捷操作按钮在右上角而非底部 — 地图场景下卡片可能被部分遮挡，操作入口要在首屏可见区
- 计划量摘要在点位详情上方 — 先看全局数量，再看具体点位
- 底部操作入口用 icon + 文字 — 跳转到关联记录页，不是对当前卡片的操作
- 调入卡片可能带附加任务提示 — 到了目的地后还能顺便做保养/清洗

## 换电任务卡片

**适用对象**：换电任务

**信息层级**：
1. 识别层：任务类型标签（换电 + 具体类型如「0电换电-紧急」）+ 右侧状态
2. 车辆信息：编号（可复制）+ 右侧电量显示（电池图标 + 百分比）
3. 标签：故障标签（如「超区」「失联」）
4. 执行信息：执行运维 + 时间
5. 操作层：分割线 → 操作按钮（待处理有）

**组织范式**：
- 电量信息右对齐，与左侧编号形成对比，一眼看到车辆状态
- 指派任务在编号前加「指派」Tag
- 卡片高度根据状态变化 — 有操作按钮时更高，无操作时更紧凑

---

## 车品任务卡片

**适用对象**：车品维修/保养任务

**信息层级**：
1. 识别层：任务类型标签（车品 + 具体类型如「车辆需维修」）+ 右侧状态
2. 车辆信息：编号（可复制）
3. 标签：故障类型标签
4. 执行信息：执行运维 + 时间

**组织范式**：
- 结构最简洁，因为车品任务信息量少
- 无地址信息（车品任务不需要导航到车辆位置）

---

## 车辆卡片（单车/助力车）

**适用对象**：车辆列表中的单个车辆

**信息层级**：
1. 核心信息：编号（可复制）+ 右侧电量（助力车有，单车无）
2. 属性信息：车龄 | 车型（竖线分隔）
3. 标签：状态标签（60+闲置、建筑区闲置等）
4. 操作栏：三等分底部操作（车辆检测 | 响铃找车 | 导航定位），用竖线分隔

**组织范式**：
- 底部操作栏是车辆卡片独有的 — 运维人员看到车辆后的三个高频操作直接暴露
- 三个操作等权重等分，用 icon + 文字，不区分主次
- 助力车比单车多一个电量信息，放在编号右侧

---

## 车牌卡片（小卡片）

**适用对象**：车牌管理列表中的单个车牌

**信息层级**：
1. 类型标签 + 右侧状态（正常/暂停/未绑定）
2. 车牌号（大字突出）
3. 交管所车牌（小字辅助）

**组织范式**：
- 小尺寸卡片（158×96），适合网格排列
- 信息极简，只有识别和状态两层

---

## 带车牌的车辆卡片

**适用对象**：租车场景中的车辆

**信息层级**：
1. 核心信息：车牌号 + 电量 + 右侧状态（可租|正常）
2. 分割线
3. 地址信息：位置 + 导航入口
4. 时间信息：拟合位置时间
5. 标签：业务标签

**组织范式**：
- 车牌号作为主标识（比编号更直观）
- 地址和时间帮助运维判断车辆是否在预期位置

---

## 运维人员卡片

**适用对象**：运维人员列表

**信息层级**：
1. 核心信息：姓名 + 右侧箭头（可展开/进入详情）
2. 归属信息：归属网格组
3. 目标与进度：清理目标量 + 已完成量 + 进度条（已开始执行时）

**组织范式**：
- 未开始执行时：目标量 + 「开始」按钮
- 已开始执行时：目标量 + 已完成量 + 进度条百分比
- 进度条是运维人员卡片独有的 — 管理者需要一眼看到每个人的完成进度

---

## 租车订单卡片

**适用对象**：租车订单

**信息层级**：
1. 识别层：状态（待使用/已完结/已取消）+ 右侧类型（立即用车）
2. 时间线：开始时间 → 结束时间（圆点 + 竖线连接）
3. 分割线
4. 订单编号 + 复制

**组织范式**：
- 时间线用圆点 + 竖线表达时间跨度，比纯文字更直观
- 订单编号在底部，因为它是辅助信息（用户先看状态和时间）

---

## 调度派单计划卡片

**适用对象**：派单计划

**信息层级**：
1. 识别层：类型标签（派单 + 指派方式）+ 右侧状态（正在派单）
2. 方向标记：出/入标签 + 竖线连接
3. 点位信息：站点名称 → 地址 + 导航 → 标签 → 等级 + 已派量 + 已领取（竖线分隔）
4. 数量信息：待派量 + 起派量（带背景色块突出）
5. 执行信息：驻点运维 + 身份 Tag
6. 分割线
7. 时间线：派单开始/结束时间 + 操作按钮

**组织范式**：
- 信息最丰富的卡片类型，因为派单计划涉及点位、人员、数量、时间多个维度
- 待派量/起派量用背景色块突出，这是运维最关心的「还要派多少」
- 时间线放在底部，因为它是约束条件而非核心操作对象
