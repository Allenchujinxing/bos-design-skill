# 经营看板（Operation Overview）

> 本文件持续补充中。

## 适用场景

全职运营人员需要查看区域运营数据、实时车辆状态、订单趋势，做出运营决策。不需要对具体对象逐个操作，核心是「看数据 → 发现问题 → 跳转到对应工具处理」。

## 需求目标

**用户目标：** 快速了解当前区域的运营健康度，发现异常指标，决定下一步行动。

**产品目标：** 让运营人员在一个页面内获得全局视角，减少在多个工具间跳转查数据的成本；通过趋势对比暴露问题，驱动主动干预。

## 首屏优先级

- **必须出现**：区域上下文（当前城市/街道）+ 高频工具入口 + 核心指标概览
- **可下滑**：更多指标分组、图表、详细数据

## 框架设计

页面分为三个大区域：

1. **顶部导航区**（固定）：城市切换 + 业务线切换（单车/助力车）
2. **工具入口区**：高频工具图标网格（4列），带「全部工具」入口
3. **数据内容区**（可滚动）：Tab 切换不同数据视角（运营表现 / 任务监控）

数据内容区内部结构：
- 分组标签（运营表现 / 执行表现）+ 时间范围选择
- 指标分类 Tab（全部 / 订单 / 资产 / 调度 / 换电）
- 指标网格（3列 × N行）

## 信息组织

### 信息层级

1. **最高优先级 — 区域上下文**：当前在看哪个城市/街道，固定在顶部。用户必须始终知道数据范围。
2. **第二优先级 — 工具入口**：高频操作的快捷入口，减少跳转层级。
3. **第三优先级 — 核心指标**：运营表现的关键数字（订单数、可用车、出单率等），大字突出。
4. **第四优先级 — 趋势对比**：较昨日/较上周的变化百分比，辅助判断趋势方向。

### 组织范式

#### 双视角 Tab 切换

「运营表现」和「任务监控」是同一个页面的两个视角。运营表现看指标数据，任务监控看执行情况。用顶部 Tab 切换，不跳页。

#### 指标网格 3列等宽排列

每个指标格子包含：label（指标名）→ value（数值，大字）→ 趋势（较昨日 +/-）。3列等宽让用户横向扫描对比。

#### 指标分类 Tab 横向滚动

当指标维度多时（全部/订单/资产/调度/换电/资保），用横向 Tab 切换分类，不是全部平铺。减少纵向滚动，用户按关注的业务线查看。

#### 时间范围可切换

右上角提供时间范围选择（今天/自定义），数据随之刷新。更新时间戳始终可见，让用户知道数据的时效性。

#### 工具入口嵌入首屏

高频工具（AI助手、工单中心、排班、找车任务）放在数据区上方。用户看完数据后的下一步通常是去某个工具操作，不需要回到首页再找入口。

#### 经营助手：数据 + 图表组合

实时数据用指标网格展示（3列×3行），历史趋势用分时折线图展示。图表支持今日/昨日/上周三条线对比，用户一眼看到趋势差异。

## 交互与状态

### 关键交互

- Tab 切换：运营表现 / 任务监控视角切换
- 指标分类 Tab：横向滑动切换业务维度
- 时间范围选择：点击触发底部弹层选择日期
- 指标说明：点击「指标说明」触发底部弹层展示公式
- 工具入口点击：直接跳转对应功能页
- 城市/街道切换：点击顶部城市名触发 Selector

### 状态覆盖

- 默认：展示当天实时数据
- 加载中：指标格子 skeleton
- 数据异常/预警：指标值变色（如完成率过低变红）+ 趋势 Tag 标红
- 无数据：指标显示「--」

## Do / Don't

- **Do**：核心指标大字突出，趋势对比小字辅助
- **Do**：时间范围和更新时间始终可见
- **Do**：工具入口放在首屏，减少跳转
- **Do**：指标按业务分类，不要全部平铺
- **Don't**：不要把经营看板做成对象操作页；若需要立即干预，先判断它是工具跳转、指标下钻还是少量高价值动作
- **Don't**：不要隐藏数据更新时间，运维场景下数据时效性很重要
- **Don't**：不要让图表占据太多首屏空间，指标数字比图表更重要
