# 信息录入（Data Entry）

## 适用场景

运营人员需要创建计划、配置规则、指派任务、修改指派关系时，进入独立全屏表单页面填写和提交。典型场景：创建目标任务、新增投车指派、修改指派、替换运维、批量修改配置。

## 需求目标

**用户目标：** 按照业务规则填写必要信息，完成创建/修改/指派操作。

**产品目标：** 确保用户填写完整且正确的信息后才能提交；通过分组和引导降低填写门槛；关键操作需要确认防止误操作。

## 首屏优先级

- **必须出现**：Title Bar（标题 + 返回）+ 操作说明（如有）或第一个表单分组
- **可滚动**：更多表单字段、关联数据结果

## 框架设计

1. Status Bar
2. Title Bar（标题 + 返回按钮）
3. 表单内容区（可滚动）
4. 到底提示
5. 底部操作栏（固定，单按钮或双按钮）
6. Home Indicator

## 信息组织

### 1. 先判断是否需要操作说明

操作说明不是信息录入页的默认头部模块。常规录入表单应让用户直接进入第一个表单分组；仅当表单存在明显步骤依赖、批量影响或较高规则理解成本时，才在填写前用步骤引导（圆点 + 竖线连接）说明整体流程。

### 2. 数据联动即时反馈

选择某个字段后，关联数据立即在同页面展示。例如选择运维后，带出该运维负责的点位列表；选择网格后，带出符合条件的点位。不跳转页面，让用户在提交前看到完整影响范围。

### 3. 字段布局先服从填写体验

录入型字段默认先考虑上下布局，让 label、placeholder、输入值和长选中结果有足够空间。左右布局适合字段值短、主要任务是快速扫读或轻配置的场景；不要为了压缩高度牺牲填写理解。系统已带出但仍允许用户编辑的值，仍按录入字段参与同一填写流，不默认降级成只读摘要或改用另一套布局语法。Placeholder 足以说明输入内容时，不额外添加 tips/helper；helper 优先留给校验、格式约束、风险说明或必须持续可见的规则。

### 4. 层级选择按承载量分流

选择类字段先判断层级深度、末级内容量和用户是否需要浏览比较。单层短选项可留在表单内快速点选；多层级选择，尤其末级选项较多、需要看清路径或确认层级时，优先进入独立选择流程承载浏览、展开收起和确认，再回填结果，不默认把复杂决策压进当前表单控件。

### 5. 字段图标按扫描需要使用

户外场景或字段类型差异需要强化时，字段可用 icon + label 形成扫描锚点。图标不是每个表单字段的默认装饰，能靠 label 和结构清楚表达时保持克制。

### 6. 只读信息与可编辑信息明确区分

系统预设的参数（如定时派单时间、任务时效）单独成区，标注「不可更改」，视觉上降低权重。用户一眼能分清哪些能改、哪些不能改。

### 7. 数据摘要嵌入表单

在表单字段之间嵌入数据摘要条（如网格闲散车数量、运维人数），让用户在填写时能看到决策依据，而不是填完再去别的页面查数据。若摘要直接由某个字段选择触发，优先与该字段保留同一模块归属，可用模块内分割或紧邻摘要区表达反馈关系，不默认拆成独立卡片。

### 8. 分组靠留白不靠分割线

表单字段按业务逻辑分组，组间用间距区隔。不用分割线，保持页面干净。同组内的字段紧凑排列，组间留出明显间距。

### 9. 底部操作固定

提交/取消按钮固定在底部，填完随时可提交，不需要滚到页面底部才能找到按钮。仅在提交前需要用户知晓的全局规则或风险提示，优先靠近底部操作区，而不是抬成独立表单模块。

### 10. 顶部导航不加背景色

导航栏和内容区融为一体，不通过背景色强调导航自身存在。视觉上减少一层分隔，用户注意力直接聚焦在表单内容上。

### 11. 可选 Tag 根据层级选择尺寸

表单中的可选标签根据所在层级选择尺寸：模块内（如表单字段中的循环周期选择）用 32px 高度，页面级别（如弹层里的标签选项）用 40px 高度。同一层级内统一使用同一种尺寸，不混用。

## 交互与状态

### 关键交互

- 输入聚焦：点击字段弹出键盘，显示清除按钮
- Picker 触发：点击日期/时间/选择类字段，弹出底部 Picker 弹层
- 添加人员/对象：点击「点击添加」跳转选人页面或弹层，选完回填
- Tag 多选：点击 Tag 切换选中/未选中（如循环周期选择周几）
- 时段滑杆：拖动双滑块选择起止时间范围
- 数据联动：选择关键字段后，自动查询并展示关联数据
- 提交校验：必填字段未填时提示，校验通过后提交
- 双按钮操作：取消（返回）/ 确认（提交）

### 状态覆盖

- 默认：所有字段为空/placeholder 状态
- 填写中：键盘弹起，当前字段聚焦
- 已填写：字段显示已输入值，关联数据已带出
- 校验错误：字段标红 + 错误提示
- 提交中：按钮 loading
- 提交成功：toast 提示 + 返回上一页

## Do / Don't

- **Do**：常规录入先让用户进入字段，复杂操作再展示必要的操作说明
- **Do**：选择后立即展示关联数据，让用户确认影响范围
- **Do**：先按填写体验选择上下或左右布局，再落到具体 Input 变体
- **Do**：预填但可编辑的字段继续按录入字段设计，保持同一填写流的节奏
- **Do**：多层且末级内容量大的选择任务优先给独立选择流程承载
- **Do**：字段图标按扫描需要使用，不把 icon 当默认装饰
- **Do**：必填字段明确标注 `*`
- **Do**：只读信息明确标注「不可更改」
- **Do**：联动摘要靠近触发字段并保持模块归属，辅助用户决策
- **Don't**：不要把所有字段平铺不分组
- **Don't**：不要给能由 placeholder 清楚表达的常规字段默认添加 tips/helper
- **Don't**：不要把系统带出的可编辑字段误做成只读摘要
- **Don't**：不要把需要浏览层级和路径的复杂选择默认塞回当前表单控件
- **Don't**：不要隐藏关联数据，用户需要在提交前看到完整影响范围
- **Don't**：不要让提交按钮跟随滚动消失
- **Don't**：不要在表单页里嵌套太多弹层，保持主流程在一个页面内完成
- **Don't**：不要让用户填完表单后还要去别的页面查数据来验证
