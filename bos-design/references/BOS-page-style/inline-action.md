# 页内辅助操作（Inline Action）

> 本文件持续补充中，后续会加入更多子类型。

## 适用场景

用户在当前页面需要进行辅助操作（筛选、配置、选择、查看详情、上报）时，不离开当前页面，通过底部弹层完成。遮罩聚焦用户注意力到弹层内容。

## 需求目标

**用户目标：** 在不丢失当前页面上下文的情况下，快速完成一个辅助决策或配置操作。

**产品目标：** 减少页面跳转，保持用户在主任务流中；通过遮罩引导用户聚焦当前操作，避免误触背景内容。

## 首屏优先级

- **必须出现**：弹层标题（说明当前操作是什么）+ 第一组可操作内容
- **可滚动**：更多选项/分组

## 框架设计

所有底部弹层共享同一套骨架：

- 背景层：黑色半透明遮罩
- 固定顶部：弹层标题栏（标题文字，部分含关闭按钮）
- 内容区：根据子类型变化（可滚动）
- 固定底部：Toolbar（确认/取消）+ Home Indicator

弹层高度由内容决定，但不超过屏幕 85%。保留顶部露出，暗示用户可以关闭回到主页面。

## 信息组织

### 信息层级

1. **最高优先级 — 标题**：始终告知用户当前在做什么操作，固定在弹层顶部不随内容滚动。
2. **第二优先级 — 可操作内容**：弹层存在的核心目的，用户需要在这里做选择或查看信息。
3. **第三优先级 — 确认操作**：固定底部，与标题形成「头尾闭合」，选完即可确认。

### 组织范式

#### 内容按业务分组，用分组标签区隔

当弹层内选项多时（如自定义数据指标），按业务维度分组（订单/资产/调度/换电），每组用带竖线装饰的分组标签标识。用户快速定位到自己关心的分组。

#### 选项用网格排列，最大化横向利用

chip/选项用 3 列网格排列，减少纵向滚动。移动端屏幕宽度有限但用户不喜欢在弹层里大量滚动。

#### Picker 类弹层固定高度

日期/时间选择用滚轮交互，需要稳定的操作区域。开始/结束双栏并排展示，用户一眼看到完整时间范围。

#### 功能入口用图标网格

「更多工具」类弹层用 4 列图标网格（icon 32px + 文字），按业务线分组。点击直接跳转，不需要确认按钮。

#### 信息详情用分组卡片 + label-value 布局

查看详情类弹层（审批信息、订单详情）用分组标签区隔不同信息块，每块内用 label-value 左右对齐布局。时间流转用时间轴组件展示。

#### 拍照上报保持极简

抽检类弹层只放上传区域 + 提示信息，弹层高度小（约 40%），操作聚焦在拍照这一件事上。

## 子类型

### 1 — 多选配置型
- **场景**：用户自定义要展示的数据指标
- **设计思路**：按业务分组 → 每组下 chip 多选 → 确认提交。内容多需要滚动，分组标签帮助定位。

### 2 — 排序配置型
- **场景**：用户选择列表的排序维度和方向
- **设计思路**：两步决策（选什么 + 怎么排），用两个分组区隔。chip 单选，选完即可确认。

### 3 — 日期/时间选择型
- **场景**：用户选择日期范围或时间范围
- **设计思路**：开始/结束双栏并排展示当前选择 → Picker 滚轮操作 → 可选「全天」Switch 简化操作。
- **变体**：纯日期 / 日期+时间 / 全天开关

### 4 — 功能入口型
- **场景**：展示所有功能模块入口
- **设计思路**：按业务线分组（通用/调度/换电）→ 图标网格 4 列排列 → 点击直接跳转，无需确认。

### 5 — 简单列表选择型
- **场景**：从短列表中选择一个选项
- **设计思路**：标题 + 列表 + 底部按钮，结构最简单。列表项少时不需要搜索和分组。

### 6 — 审批/审核详情型
- **场景**：查看审批申请的详细信息和流程状态
- **设计思路**：信息分两块（申请信息 + 审批流程），用分组标签区隔。申请信息用 label-value 布局 + 图片预览，审批流程用时间轴展示节点。状态 Tag 标识当前进度。不同状态下 Toolbar 操作不同（待审核有通过/拒绝，已完成有知道了）。

### 7 — 拍照上报/抽检型
- **场景**：流程中触发抽检，用户需要拍照上传凭证
- **设计思路**：极简聚焦 — 只放上传区域 + 可选的提示通知栏。弹层高度小（约 40%），核心操作就是拍照这一件事。提交后不可撤回。

### 8 — 信息详情型
- **场景**：点击列表/地图中的对象，查看详细信息
- **设计思路**：信息按卡片分组（状态+时间线、金额、服务信息），每组有分割线。服务信息用 icon + label 左对齐、value 右对齐的行布局。关键 ID 支持复制。底部单一操作按钮。

## 交互与状态

### 关键交互

- 打开：从底部滑入，背景加遮罩
- 关闭：点击遮罩 / 下滑弹层 / 点击关闭按钮 / 点击确认
- Chip 选择：单选高亮切换 / 多选 toggle
- Picker 滚动：滚轮选择日期/时间
- 确认：点击 Toolbar 按钮提交选择

### 状态覆盖

- 默认：展示所有可选项，无选中态
- 已选中：chip 高亮 / Picker 停在选中值
- 加载中：弹层内容 skeleton
- 无可选项：提示文案

## Do / Don't

- **Do**：弹层高度不超过 85% 屏幕，保留顶部露出感
- **Do**：标题明确说明当前操作目的
- **Do**：内容多时按业务分组，用分组标签区隔
- **Do**：确认按钮始终可见（固定底部）
- **Don't**：不要在弹层里嵌套弹层
- **Don't**：不要让弹层内容需要横向滚动
- **Don't**：不要省略遮罩，用户需要明确知道这是一个临时操作层
- **Don't**：不要在弹层里放太多步骤，超过 2 步应该用独立页面
